# 実装タスク一覧

## 1. ディレクトリと空ファイルの準備
- [ ] **プロジェクト構造の作成**
    - `janken_app_js` フォルダ内に以下のファイルを空で作成する。
    - `index.html`
    - `style.css`
    - `script.js`

## 2. UI（HTML/CSS）の実装と基本配置
- [ ] **HTML構造の作成 (`index.html`)**
    - タイトルタグの設定。
    - ボタン用エリア（ID: `buttons`）の設定。
    - 結果表示エリア（ID: `result-area`）の設定。
- [ ] **CSSによるスタイリング (`style.css`)**
    - 全体的なレイアウト（中央揃えなど）を整える。
    - ボタンと結果エリアの見た目を調整する。

## 3. ロジック（JavaScript）の基本実装
- [ ] **ゲームロジックの強化 (`script.js`)**
    - 現在の対戦数（最大5回）を管理する変数 `matchCount` を定義。
    - 勝ち数、負け数、あいこ数を管理するスコア変数 `wins`, `losses`, `draws` を定義。
    - コンピュータの手を決定する関数 `getComputerChoice()` を作成。
    - 勝敗を判定する関数 `determineWinner(user, computer)` を作成。

## 4. 画面とロジックの結合（ボタン連携・スコア表示）
- [ ] **イベントリスナーとスコア更新の実装 (`script.js`)**
    - HTMLのボタンクリック時に `playGame()` 関数を呼び出す。
    - 5回未満であれば、判定結果と累計スコアを画面に表示する。
    - 5回終了時には、対戦ボタンを無効化し、最終結果を表示する。

## 5. 動作確認（手動テスト）
- [ ] **ブラウザでの動作確認**
    - `index.html` をブラウザで開き、ボタンをクリックして動作を確認する。
    - ユーザーの手、コンピュータの手、勝敗が表示されるか確認する。
- [ ] **勝敗判定の確認**
    - 全ての組み合わせ（勝ち・負け・あいこ）が正しく判定されるかを確認する。
