# テスト仕様書：ポケモン情報表示アプリ

## 1. 目的
本テスト仕様書は、PokeAPIを用いたポケモン情報表示機能が、要件定義および設計仕様に基づき、意図通りに動作することを網羅的に検証することを目的とする。

## 2. 検証項目

| No | 検証内容 | 手順 | 期待結果 | 結果（机上デバッグ） | 結果（手動テスト） |
|---|---|---|---|---|---|
| 1 | 正常系：ピカチュウ情報の取得と表示 | アプリを起動する | ピカチュウの画像と名前が表示される | OK | - |
| 2 | 正常系：レンダリング時のAPI重複呼び出しなし | 画面をレンダリングし、コンソールを確認する | API取得が一度のみ行われること | OK | - |
| 3 | 異常系：通信エラー発生時の挙動 | APIのURLを無効なものに書き換える | エラーメッセージが表示される | OK | - |
| 4 | 異常系：読み込み中の表示 | APIのレスポンスを遅延させる（または通信状態を制限する） | 「Loading...」と表示される | OK | - |

## 3. 静的解析および机上デバッグ結果の根拠

### 3.1. 正常系：ピカチュウ情報の取得と表示
- `PokemonCard.tsx` 内で `useFetchPokemon` を呼び出し、返り値の `data` が存在する場合に `data.name` と `data.sprites.front_default` をレンダリングしているため、データ取得後に正しく表示されます。

### 3.2. 正常系：レンダリング時のAPI重複呼び出しなし
- `useFetchPokemon.ts` 内で `useRef` を用いて `isMounted` フラグを管理し、`useEffect` 内で `if (!isMounted.current) { isMounted.current = true; ... }` の条件分岐を行っているため、再レンダリング時でもAPIリクエストは初回のみ実行されます。

### 3.3. 異常系：通信エラー発生時の挙動
- `useFetchPokemon.ts` の `fetchData` 関数内で `try-catch` ブロックを使用し、API呼び出しが失敗した際に `setError` を呼び出しています。`PokemonCard.tsx` では `error` ステートが設定されている場合にエラー表示（`Error: ...`）を行うため、正常にハンドリングされます。

### 3.4. 異常系：読み込み中の表示
- `useFetchPokemon.ts` で `loading` ステートの初期値を `true` とし、データ取得成功または失敗時に `setLoading(false)` を呼び出します。`PokemonCard.tsx` は `loading` が `true` の間に「Loading...」を表示するため、正しく読み込み中状態が表現されます。
