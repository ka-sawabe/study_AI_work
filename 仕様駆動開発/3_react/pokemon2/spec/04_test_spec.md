# テスト仕様書：ポケモン情報表示機能検証

## 1. 目的
本検証は、PokeAPIからピカチュウの情報を取得し表示する機能が、要件定義書および設計仕様書に従って正しく動作することを保証することを目的とします。

## 2. 検証項目

| No | 検証内容 | 手順 | 期待結果 | 結果（机上デバッグ） | 結果（手動テスト） |
|:---|:---|:---|:---|:---|:---|
| 01 | 正常系：情報表示 | ページを開く | ピカチュウの画像と名前が表示される | OK | OK |
| 02 | 正常系：ロード表示 | ネットワークを遅延させる | 取得中にローディングが表示される | OK | OK |
| 03 | 異常系：APIエラー | APIのURLを無効にする | エラーメッセージが表示される | OK | OK |
| 04 | 正常系：キャッシュ | ページを再表示する | 再通信せずキャッシュから表示される | OK | OK |

## 3. 論理的根拠（机上デバッグ結果）
- **01. 正常系**: `useQuery` が成功時 `data` を返し、`data.sprites.front_default` と `data.name` が存在するためレンダリングされる。
- **02. ロード表示**: `isLoading` が `true` の間、`<div>読み込み中...</div>` がレンダリングされる。
- **03. APIエラー**: `fetch` でエラーを投げ、`useQuery` が `error` オブジェクトを返すため、条件分岐によりエラーメッセージが表示される。
- **04. キャッシュ**: `TanStack Query` のデフォルト設定で成功データはキャッシュされ、`queryKey` が同一であれば再通信はスキップされる。
